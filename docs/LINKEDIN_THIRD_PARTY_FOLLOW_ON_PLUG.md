# LinkedIn follow-on plug: third-party extensions page

Low-key plug: I’ve been building an open-source monitoring + discovery tool that:
- tracks core + community extensions daily
- generates Markdown/CSV/Excel reports
- persists a DuckDB snapshot for ad‑hoc querying

New (and no longer “in-flight”): the beginnings of a **third-party extensions report** are now live.

This is a first cut at surfacing extensions discovered outside the official registries, based on automated discovery plus lightweight manual labelling. It’s explicitly a **work in progress** — partial coverage, and you should do your own due diligence before relying on anything listed.

If you’re an extension author (or you maintain an internal extension you’ve open-sourced), I’d love your participation:
- check the third-party page and see whether your extension is present and correct
- submit corrections or additions via pull request (preferred), or send a note if you’d rather not

Main report (Core & Community): https://mjboothaus.github.io/duckdb-extensions-analysis/
Third-party extensions: https://mjboothaus.github.io/duckdb-extensions-analysis/third-party/

Submissions guide: https://github.com/Mjboothaus/duckdb-extensions-analysis/blob/main/docs/THIRD_PARTY_EXTENSIONS_SUBMISSIONS.md
